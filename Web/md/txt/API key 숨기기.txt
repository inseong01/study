- 문제
클라이언트에서 api 요청을 하면 페이지에 key가 보인다.

- 해결
클라이언트에서 서버로 api key를 요청해서 key를 받아온다.
페이지에서 key가 보이지 않는다.

- 문제
클라이언트에서 서버 주소가 보인다.
누군가 클라이언트에 남아있는 주소로 요청하면 key를 받을 수 있다.

- 해결
프록시 서버, 엔드포인트 설정해서 원본 출처를 숨길 수 있다.

- 문제
프록시 서버, 엔드포인트 주소가 보인다.
누군가 클라이언트에 남아있는 주소로 요청하면 key를 받을 수 있다.


- 서버리스
key를 위탁하고 새롭게 받은 API URI로 해결가능
키는 숨겼지만 클라이언트에 URI가 남아 있어서 다른 사람도 사용할 수 있음
제품) 클라우드플레어 워커스

- JWT 토큰
로그인 정보를 토큰으로 받아 올바른 요청에만 API 제공
다만 로그인이 필요없는 서비스면 API 제공 안 됨

- IP 제한
API 요청이 비정상적으로 많으면 API 접근 금지하도록 해당 IP 등록
그나마 괜찮은 방법
서버 필요, 접근 IP 필요, express의 req.ip 확인 가능
